### 子模块 1.1：线性代数——数据在高维空间里的"搬家"

**学习目标**：不要求会手算复杂的矩阵分解，但要能**看懂**矩阵乘法、转置、逆矩阵在干什么。

| 知识点 | 大白话解释 | 学了能干啥 |
|:---|:---|:---|
| **标量、向量、矩阵、张量** | 标量 = 一个数字（温度 36.5°C）；向量 = 一排数字（体检单上的各项指标）；矩阵 = 一张表格（全班同学的体检单）；张量 = 一摞表格（各班各次体检数据叠起来） | AI 里所有数据都以"张量"形式流动 |
| **矩阵乘法（点积）** | 就是"加权求和"——你把 N 个输入按不同权重累加起来，得到 M 个输出。相当于做菜：每种食材（输入）按不同比例（权重）下锅，炒出几道菜（输出） | 神经网络的一层，本质上就是一次矩阵乘法 |
| **转置** | 把表格的行和列互换——横着看变成竖着看，方便和另一个矩阵相乘 | 调整数据形状以匹配计算规则 |
| **范数（L1/L2）** | L2 范数 = 勾股定理推广到高维——一个点到原点的"直线距离"。L1 = 只能横竖走的"曼哈顿距离" | L2 范数用于衡量模型预测值和真实值的差距 |
| **特征值与特征向量** | 一个矩阵对一个向量的作用，有些向量方向不变只是被拉伸（特征向量），拉伸的倍数就是特征值 | PCA 降维、PageRank、图神经网络的基础 |

**实战任务**：
1. 用 NumPy 手写一个 2×3 矩阵和一个 3×2 矩阵相乘，用纸笔先算一遍再对照代码验证
2. 用 Matplotlib 画一个二维向量 `[3, 2]`，再画它被矩阵 `[[2, 0], [0, 0.5]]` 变换后的结果，直观感受"矩阵 = 空间变换"
3. 手写代码实现"用 L2 范数（欧氏距离）找一个点最近的邻居"

**🎯 首选资源**：

### 子模块 1.2：微积分——下山的最陡方向

**学习目标**：理解 **"梯度 = 函数下降最快的方向"** 这一个核心概念就够了。

| 知识点 | 大白话解释 | 学了能干啥 |
|:---|:---|:---|
| **导数** | 你在高速上开车，导数就是仪表盘上的瞬时速度——"这一刻你变化有多快" | 理解参数更新的速率 |
| **偏导数** | 一个函数有多个旋钮（参数），偏导数就是"只拧一个旋钮，其他不动，看看输出怎么变" | 神经网络有百万个参数，每次只关心一个 |
| **梯度** | 所有偏导数组成的向量——指向函数值**上升最快**的方向 | 梯度下降：你往梯度的**反方向**走，就能最快下山（让损失函数变小） |
| **链式法则** | 多米诺骨牌：A 变化导致 B 变化，B 变化导致 C 变化。链式法则告诉你 A 对 C 的最终影响 = 各段影响相乘 | 反向传播（BP）的数学根基 |

**实战任务**：
1. 用纸笔计算 `f(x, y) = x² + y²` 的梯度，然后用 NumPy 在 `x ∈ [-5, 5], y ∈ [-5, 5]` 范围内做网格采样，画一组"梯度箭头图"（quiver plot），直观看到所有箭头都指向 (0, 0)
2. 纯手写一个**一维梯度下降**：目标函数 `f(x) = (x - 3)²`，从 `x = 10` 出发，每次往反方向走一小步，打印每次的 x 值，观察收敛到 3 的过程

**🎯 首选资源**：
- **3Blue1Brown 的《微积分的本质》**——第 8—10 集（偏导数与梯度）。同样动画讲解，看完就知道"为什么要往反方向走"。

### 子模块 1.3：概率论——面对不确定性的数学工具

**学习目标**：理解 AI 为什么"概率化"地去处理世界，掌握贝叶斯思维的雏形。

| 知识点 | 大白话解释 | 学了能干啥 |
|:---|:---|:---|
| **随机变量与分布** | 掷骰子——结果是随机的，但你大概知道每种结果出现的比例（分布） | 描述世界的不确定性 |
| **条件概率** | "已知下雨了，堵车的概率是多少？"——这就是条件概率 | 贝叶斯分类器、朴素贝叶斯的基础 |
| **贝叶斯公式** | 你看到地上湿了（证据），反推刚才可能下过雨（原因）。公式告诉你如何用观察到的东西反推未知的原因 | 整个生成式 AI 的底层思想 |
| **期望、方差、协方差** | 期望 = "平均大概多少分"；方差 = "成绩波动大不大"；协方差 = "数学考得好的人，物理是不是也考得好" | 理解损失函数、正则化、特征选择 |
| **最大似然估计（MLE）** | 你看到一堆数据，猜"什么样的参数最可能生成这些数据" | 损失函数的推导源头（交叉熵 ≈ 最大似然） |
| **交叉熵** | 两个概率分布之间"有多不像"的度量——好比你说"是猫"，真实标签说"是狗"，差距越大，交叉熵越大 | 分类问题的默认损失函数 |

**实战任务**：
1. 用 NumPy 生成 10000 个服从正态分布的随机数，画直方图并叠加理论概率密度曲线，直观感受"大数定律"
2. 用纯 NumPy 实现一个简单的**高斯朴素贝叶斯分类器**（用鸢尾花数据集的前两个特征），不调 sklearn

**🎯 首选资源**：

### 子模块 2.2：NumPy——AI 数据管的"高速公路"

**学习目标**：NumPy 不是"一个库"，而是**AI 世界的空气**——你在它上面呼吸，但你感觉不到它的存在。要把 NumPy 操作练到"肌肉记忆"的程度。

| 知识点 | 大白话解释 | 学了能干啥 |
|:---|:---|:---|
| **ndarray 的 shape / dtype / stride** | shape = 这个张量长什么样（几行几列）；dtype = 每个格子存储的是什么类型；stride = 在内存中跨几步到下一个元素 | 理解为什么某些操作快、某些慢 |
| **广播机制（Broadcasting）** | 两个形状不同的数组怎么自动"对齐"——有点像 Excel 的填充柄 | 不写 for 循环就能让不同形状的数据做运算 |
| **切片与花式索引** | 切片 = 切蛋糕（取子矩阵）；花式索引 = 按清单挑（取第 1, 5, 9 行） | 数据批处理、采样 |
| **轴（axis）的概念** | axis=0 = "从上到下"沿着行方向操作（跨行）；axis=1 = "从左到右"沿着列方向操作（跨列）。sum(axis=0) = 把每一列的行的值加起来 = "把行压扁" | 理解 `np.sum`、`np.mean` 等聚合操作的方向 |
| **向量化编程思维** | 一次操作整个数组，而不是用 for 循环一个元素一个元素地处理 | 代码提速 10—100 倍 |

### 子模块 3.1：张量操作——PyTorch 里的 "NumPy"

**学习目标**：把模块二中 NumPy 的所有操作在 PyTorch 里重做一遍，感受两套 API 的高度相似性。

| 知识点 | 大白话解释 | 学了能干啥 |
|:---|:---|:---|
| **`torch.tensor` vs `np.ndarray`** | PyTorch 的张量 = NumPy 数组 + **梯度记录开关**。你把开关打开（`requires_grad=True`），后面的所有计算都会被自动记录下来 | PyTorch 唯一的数据结构，所有模型、数据、参数都在里面 |
| **`torch.from_numpy()` / `.numpy()`** | NumPy 数组和 PyTorch 张量共享同一块内存，就像同一个文件柜从两个门打开——不是"拷贝"而是"视图" | CPU 上无缝切换两种格式 |
| **`torch.cuda` 与设备管理** | 张量可以放在 CPU 上（默认），也可以搬到 GPU 上（`x.to('cuda')`）。GPU 上的张量比 CPU 快几十倍——但 GPU 显存有限 | 第一次接触"设备"概念 |
| **`view` / `reshape` / `permute`** | view = 重新捏形状（不改变数据本身）；permute = 转置维度（把 RGB 从 (H, W, C) 变成 (C, H, W)） | 所有神经网络都需要在层与层之间调整张量形状 |

### 子模块 3.2：自动求导——PyTorch 的"灵魂功能"

**学习目标**：理解 `loss.backward()` 这行代码背后到底发生了什么。

| 知识点 | 大白话解释 | 学了能干啥 |
|:---|:---|:---|
| **计算图（Computation Graph）** | PyTorch 在你每做一次运算时，都在后台画一个"运算关系图"——谁是由谁算出来的，记录得明明白白 | `backward()` 时沿着图反向遍历 |
| **`requires_grad=True`** | 给张量贴个标签："我需要你对最终的输出负责，待会儿要算你的梯度" | 模型参数 w 和 b 必须设置为 True |
| **`loss.backward()`** | 引擎启动——从 loss 出发沿着计算图一路往回跑，用链式法则计算出每个 `requires_grad=True` 节点的梯度 | 训练循环的固定步骤 |
| **`torch.no_grad()`** | 临时关掉梯度记录——"我接下来只是看一下结果，别浪费显存记图了" | 做预测 / 评估时必须用的上下文管理器 |
| **`zero_grad()`** | 每次 `backward()` 后梯度会**累加**而不是覆盖。`zero_grad()` = 擦黑板——清空上次的梯度 | 训练循环第一步，最容易忘的一步 |

**实战任务**：
1. 给定 `x = torch.tensor([2.0], requires_grad=True)` 和 `y = x**3 + 2*x + 1`，手动算出 `dy/dx` 在 x=2 的值，再执行 `y.backward()` 看 `x.grad`，验证结果
2. 用 PyTorch 的自动求导实现一遍"模块一"里的梯度下降演示，对比手写梯度 vs `x.grad` 的结果
3. 写一个3次调用 `y.backward()` 但中间不 `zero_grad()` 的循环，观察梯度确实在**累加**，理解为什么训练循环必须 `zero_grad()`

**🎯 首选资源**：

### 子模块 3.3：第一个训练循环——麻雀虽小五脏俱全

**学习目标**：在进入秋季正式学习之前，先用最简单的例子跑通一次**完整的训练循环**，建立全局认知。

**📖 子模块详解**：什么是训练循环？它就像你学骑自行车：① 上车骑一段（前向传播，得到预测结果）→ ② 发现歪了（计算损失函数，衡量差距）→ ③ 调整车把角度（反向传播，计算梯度）→ ④ 调整身体重心（优化器更新参数）。重复几百次就学会了。这个循环是所有深度学习任务的统一框架。

**一个完整的 PyTorch 训练循环包含五个固定步骤**：
```python
for epoch in range(num_epochs):          # 重复骑车练习
    y_pred = model(X)                    # ① 前向传播：预测
    loss = loss_fn(y_pred, y_true)       # ② 计算损失：差距有多大
    optimizer.zero_grad()                # ③ 清空上次梯度（擦黑板）
    loss.backward()                      # ④ 反向传播：算出每个参数的梯度
    optimizer.step()                     # ⑤ 参数更新：往梯度反方向走一步
```
这五行代码是你整个 AI 学习生涯中会写最多遍的代码。

**实战任务**：
1. 用 PyTorch 实现**一元线性回归**：用 `nn.Linear(1, 1)`，随机生成一些 `y = 3x + 2` 附近的点，训练到 loss < 0.01，打印学到的 w 和 b（应该接近 w≈3, b≈2）
2. 在上面的代码基础上，加入 TensorBoard 可视化：画出 loss 曲线、参数的更新轨迹（用 `SummaryWriter`）

**🎯 首选资源**：

### 子模块 4.1：什么是"机器学习"——基本术语与归纳偏好（西瓜书 §1.1—§1.4）

**学习目标**：建立 ML 的"话语体系"——理解样本、特征、标签、模型、泛化等最基础的术语，理解"没有免费的午餐"定理的含义。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **样本 / 特征 / 标签** | 样本 = 一道题；特征 = 题目的已知条件（面积、卧室数）；标签 = 正确答案（房价） | §1.1 基本术语 |
| **训练集 vs 测试集** | 训练集 = 你用来学习的"题库"；测试集 = 真正考试的"真题"——学的时候不能看真题，否则就是作弊 | §1.1 + §2.2 |
| **假设空间** | 所有"可能的规律"的集合——好比所有可能的菜谱，你要从中找出最好吃的那份 | §1.2 假设空间 |
| **归纳偏好（Inductive Bias）** | 在没有绝对对错时，你偏向哪种解释——"如无必要，勿增实体"（奥卡姆剃刀）= 简单的解释优先 | §1.3 归纳偏好 |
| **没有免费的午餐定理（NFL）** | 所有算法在所有问题上的平均表现完全一样——A 算法在问题 X 上好，必然在问题 Y 上差。没有"万能算法" | §1.3（重点理解） |
| **泛化（Generalization）** | 模型在"没见过的题"上的表现——学霸不是会做题库里的题，而是会做任何新题 | §1.1 |

### 子模块 4.2：怎么判断模型"好不好"——评估方法与性能度量（西瓜书 §2.1—§2.5）

**学习目标**：这是整个 ML 课程中**最容易被低估但最实用**的一章。掌握留出法、交叉验证、各种评估指标和偏差-方差分解，你就能诊断 90% 的模型问题。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **经验误差 vs 泛化误差** | 经验误差 = 你在题库上的得分；泛化误差 = 你在真正考试上的得分。两者差距越大 = 过拟合越严重 | §2.1 |
| **过拟合 vs 欠拟合** | 过拟合 = 死记硬背答案（题库满分但新题全错）；欠拟合 = 根本没学（题库和新题都错） | §2.1（全书最核心概念之一） |
| **留出法（Hold-out）** | 把数据按比例切成训练集和测试集——最简单也最常用的评估方法 | §2.2.1 |
| **k 折交叉验证** | 把数据分成 k 份，每次用 1 份测试、k-1 份训练，轮流 k 次取平均——数据少时的救命稻草 | §2.2.3 |
| **准确率 / 精确率 / 召回率 / F1** | 四种指标的物理意义和适用场景——不要只会看准确率！类别不平衡时准确率会骗人 | §2.3.1—§2.3.2 |
| **ROC 与 AUC** | ROC 曲线画的是"真阳率 vs 假阳率"的 trade-off；AUC = 随机抽一个正例和一个负例，模型把正例排在负例前面的概率 | §2.3.3 |
| **偏差-方差分解** | 偏差 = 瞄准的准星偏不偏（模型能力够不够）；方差 = 手抖不抖（对不同训练集的敏感度）。好模型 = 准星正 + 手不抖 | §2.5（全书最重要的理论工具） |

**实战任务**：

3. **偏差-方差实验**：用不同复杂度的模型（1 阶到 10 阶多项式）拟合同一组数据，重复 100 次（每次加不同噪声），画出偏差²-方差-总误差随模型复杂度的变化曲线——亲眼看到偏差和方差的 trade-off

### 子模块 5.1：线性回归——三条路解同一个问题（西瓜书 §3.1）

**学习目标**：线性回归虽然简单，但它是整个 AI 大厦的"地基模型"。西瓜书 §3.1 给出了**三种等价视角**——最小二乘法（矩阵微积分）、梯度下降（迭代优化）、概率视角（极大似然估计）。三种视角看懂一种就能用，三种都看懂就能触类旁通。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **均方误差 MSE** | 预测值与真实值差距的平方的平均——差距越大惩罚越重（平方放大） | §3.1.1 |
| **正规方程（解析解）** | θ = (XᵀX)⁻¹Xᵀy —— 一个公式直接出答案，不需要迭代 | §3.1.2 |
| **梯度下降（迭代解）** | 沿着 MSE 对 θ 的梯度反方向，一步步走向最优解 | §3.1.3 |
| **概率视角（MLE）** | 假设误差服从正态分布 → 最大化似然 = 最小化 MSE。这条逻辑链揭示了"MSE 不是随便选的" | §3.1.4 |
| **多变量线性回归** | 从 `y=w₁x₁+b` 到 `y=w₁x₁+w₂x₂+...+b`——数学上只是矩阵维度变大，物理意义完全一致 | §3.1 |

**实战任务**：
1. **纯 NumPy 实现线性回归（三条路都走一遍）**：
   - 正规方程法：`θ = np.linalg.inv(X.T @ X) @ X.T @ y` （对应西瓜书公式 3.11）
   - 批量梯度下降法：手写梯度 = `X.T @ (y_pred - y) / m` （对应西瓜书公式 3.16）
   - 概率视角验证：用 MLE 公式推导出 θ = (XᵀX)⁻¹Xᵀy，验证与正规方程结果完全一致
2. 在加州房价数据集上对比三条路的结果和速度，画出每种方法的收敛曲线
3. 故意制造过拟合：用 10 阶多项式拟合 5 个点，画出来看到曲线对训练点"巨完美"但振荡剧烈 → 回到教材 §2.1 和 §2.5，对应过拟合和偏差-方差分解

### 子模块 5.2：对数几率回归（逻辑回归）与线性判别分析（西瓜书 §3.2—§3.4）

**学习目标**：逻辑回归虽然名字里带"回归"，但它做的是**分类**。核心技巧：在线性组合外面套一个 sigmoid 函数，把输出压缩到 0 和 1 之间。LDA 从另一个角度（类间散度最大化）做分类。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **Sigmoid 函数** | σ(z) = 1/(1+e⁻ᶻ) —— S 形曲线，把 (-∞, +∞) 压缩到 (0, 1)，天然适合表示概率 | §3.2 |
| **对数几率** | ln(y/(1-y)) = wᵀx + b —— 事件"发生/不发生"的比值的对数等于线性组合。这就是"对数几率回归"名字的由来 | §3.2.1 |
| **交叉熵损失** | 这是从极大似然估计推出的损失函数——不是随便选的！西瓜书公式 3.27 给出了完整推导 | §3.2.2 |
| **LDA（线性判别分析）** | 找一个投影方向，使得投影后"类内尽量紧凑、类间尽量分开"——相当于最大化"类间散度/类内散度" | §3.3 |
| **多分类：OvR vs OvO vs MvM** | OvR = 一对一训练多个二分类器；OvO = 两两训练；MvM（ECOC）= 用纠错码设计分类器 | §3.4 |

**实战任务**：
1. **纯 NumPy 实现逻辑回归**：包含 sigmoid 函数、交叉熵损失（对照西瓜书公式 3.27）、梯度计算、SGD 更新，在 breast cancer 数据集上分类

### 🖥️ 硬件思维 Tip #5

> **思考题**：线性回归的正规方程 θ = (XᵀX)⁻¹Xᵀy 涉及矩阵求逆——O(n³) 复杂度。当特征维度 n=10000 时，XᵀX 是 10000×10000 的矩阵，求逆需要约 10¹² 次运算。而梯度下降每次迭代只要 O(nd)（n=样本数，d=特征数）。为什么 GPU 的并行计算能力让梯度下降成为大数据时代的"王道"？（提示：矩阵求逆很难并行化，而矩阵乘法天然适合 GPU）

## 模块六：决策树——像玩"20 个问题"一样做决策（10月第1—2周）

> **对应教材**：西瓜书第 4 章（决策树）
> **核心思想**：决策树是最直观的 ML 模型——每一步问一个最优的"是/否"问题，沿着树走到叶子节点得到答案。它是唯一一个你可以把"模型"直接画出来给人看的算法。

| 维度 | 内容 |
|:---|:---|
| **周期** | 10月第1周 — 第2周（2周） |
| **每周建议用时** | 8—10 小时 |
| **📖 教材章节** | 西瓜书 §4.1—§4.5 |

**📖 子模块详解**：决策树 = 玩"20 个问题"猜人物游戏——"是男性吗？"→ 是 → "戴眼镜吗？"→ 是 → "是程序员吗？"——每一步选择**最能缩小候选范围**的提问。西瓜书第 4 章给出三种划分选择的数学标准（信息增益、增益率、基尼指数），并详细讲解剪枝策略（§4.3）和连续值处理（§4.4）。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **信息熵** | 熵 = 混乱度——如果所有标签都一样（纯），熵 = 0；正负各一半，熵最大。公式 4.1 | §4.1 |
| **信息增益（ID3）** | "用这个特征分完之后，混乱度降低了多少？"——降低越多，特征越有用。公式 4.2 | §4.2.1 |
| **增益率（C4.5）** | 信息增益偏爱"取值多的特征"（如"学号"每个值只对应一个人，混乱度降为 0 但毫无泛化）。增益率 = 信息增益 / 特征固有值，消除这种偏袒 | §4.2.2 |
| **基尼指数（CART）** | 另一种度量"纯不纯"的方式——比熵计算简单，二分类时两者效果几乎相同 | §4.2.3 |
| **预剪枝 vs 后剪枝** | 预剪枝 = 边建树边判断"再分裂有没有用"（早停）；后剪枝 = 建完再砍掉无用的分支。教材详细对比了两者的优缺点 | §4.3 |
| **连续值处理** | 年龄不是"青年/中年/老年"，而是 0—100 的连续值 → 二分法：在所有取值之间找最佳划分点 | §4.4.1 |
| **缺失值处理** | 真实数据总有缺失→ 给每个样本加权重，缺失特征的样本按权重比例分到各分支 | §4.4.2 |

**实战任务**：
1. **从零手写决策树（纯 NumPy，不调 sklearn）**：实现信息熵计算（公式 4.1）、信息增益选择最佳分裂特征（公式 4.2）、递归建树。在 Titanic 数据上训练，输出整棵树的结构（就像 if-else 语句一样可读）——对应教材 §4.1—§4.2
2. **手动实现预剪枝和后剪枝**：先跑一棵不加限制的树（过拟合严重），再分别用预剪枝（限制最大深度、最小样本数）和后剪枝（自底向上用验证集评估剪枝效果），对比三种策略的准确率和树的复杂度——对应教材 §4.3
3. 用 sklearn 的 DecisionTreeClassifier 在 digits 数据集上训练后，用 `export_text` 输出整棵树，逐条规则阅读——这个体验让你真正理解"模型可解释性"

**🎯 首选资源**：
- **西瓜书 §4.1—§4.5 + 南瓜书第 4 章公式推导**——教材对三种划分标准（ID3/C4.5/CART）的对比是中文资料里最清晰的。
- 辅助动画：**StatQuest《Decision Trees》**——B站搜索，10 分钟动画讲解决策树。

### 🖥️ 硬件思维 Tip #6

> **思考题**：决策树的训练涉及大量"按特征值排序→找最佳划分点"的操作——本质上是对每一列排序。排序是 O(n log n) 的，无法利用 GPU 的矩阵乘法加速。这是为什么决策树和随机森林在 GPU 上的加速不如神经网络明显。查一下 **LightGBM 的 histogram-based 算法**——它怎么把排序问题转化成离散化（分桶）问题来加速？

### 子模块 7.2：误差逆传播（BP）——神经网络能学习的秘密（西瓜书 §5.2—§5.3）

**学习目标**：BP 是全书最重要的算法之一。西瓜书 §5.2 给出了完整的链式法则推导（公式 5.7—5.15），南瓜书有逐行推导。目标：第一遍跟着推导走一遍，不求全部记住，但要理解"误差从输出层逐层往回传"的物理过程。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **前向传播** | 输入 → 隐藏层 → 输出，逐层计算——"正向推理" | §5.2 |
| **反向传播（BP）** | 从输出层误差出发，用链式法则逐层计算每个参数的梯度——"反向传错" | §5.2（公式 5.7—5.15） |
| **梯度消失** | sigmoid 的导数最大只有 0.25，每传一层就衰减 75%，10 层后就没了 → 这就是为什么"深度学习"在 2010 年之前无法实现 | §5.3.1 |
| **局部极小 vs 全局最小** | 损失函数的地形是凹凸不平的——梯度下降可能卡在局部最低点（局部极小）而看不到真正的最低点（全局最小） | §5.3.2 |
| **ReLU** | ReLU(x)=max(0,x)——导数要么是 0 要么是 1，不衰减！现代深度学习的默认选择 | §5.4 补充 |

**实战任务**：
1. **从零实现一个 MLP（纯 NumPy + 手动反向传播）**：2 个隐藏层，用 ReLU 激活，手写前向和反向传播。在 MNIST 子集上训练（只用 1000 张图），逐层验证你的梯度计算。核心代码对照西瓜书公式 5.7—5.15
2. 用 PyTorch `nn.Module` 实现同样的网络，打印 `model.parameters()` 里的梯度，对比手写版——验证你手写的 BP 和 PyTorch 的 autograd 算出的梯度一致
3. **实验**：同一任务分别尝试 sigmoid 和 ReLU，画 loss 曲线——sigmoid 的 loss 在深层网络中几乎不动（亲眼看到梯度消失），ReLU 快速下降

**🎯 首选资源**：
- **西瓜书 §5.2—§5.3 + 南瓜书第 5 章公式推导**（逐行推导 BP！）+ **3Blue1Brown《神经网络》第 3—4 集**（BP 的动画直觉）。

### 子模块 7.3：现代训练技巧——让网络"真的训得动"（西瓜书 §5.4 + 补充）

**学习目标**：西瓜书 §5.4 介绍了其他常见神经网络（RBF、ART、SOM、级联相关、Elman、Boltzmann 机），但这些在今天已不常用。**本模块重点转向现代深度学习训练技巧**，作为教材到实践的桥梁。

| 知识点 | 大白话解释 | 教材/来源 |
|:---|:---|:---|
| **Batch Normalization** | 在层与层之间加一道"标准化工序"——把数据拉到均值 0、方差 1，让下一层收到的数据永远是"干净"的 | 补充（对应 Ch13 神经网络的优化） |
| **Dropout 正则化** | 训练时随机关掉一部分神经元（假装它们坏了），强制网络不能依赖任何一个神经元 | 补充（对应 Ch13） |
| **Xavier / Kaiming 初始化** | 根据层的输入输出大小，自动选择初始参数的合适范围——不乱初始化 | 补充（对应 Ch13） |
| **Adam 优化器** | SGD + Momentum + 自适应步长——90% 场景的默认选择，西瓜书未涉及但在实践中缺不了 | 补充（对应 Ch13） |
| **学习率调度 + Early Stopping** | 刚开始大步快走，越接近终点步长越小；验证集不降了就停 | 补充 |
| **过拟合 vs 欠拟合 诊断** | train loss 低 val loss 高 = 过拟合；train loss 也高 = 欠拟合。回到教材 §2.1 和 §2.5 | 教材 §2.1 + §2.5 |

**实战任务**：
1. 同一个网络，对比有 BN 和无 BN 的训练过程——打印每层激活值的均值和方差
2. 画 2D 的"山谷图"（Rosenbrock 函数）：分别用 SGD、SGD+Momentum、Adam 从同一个点出发，画出优化路径
3. **综合实战**：MLP + BN + Dropout + Adam + LR Scheduler + Early Stopping，在 CIFAR-10 上达到测试集 55%+（只用全连接层）

**🎯 首选资源**：
- **吴恩达《Deep Learning Specialization》第二门课 Week 1—3**（初始化、正则化、优化的经验法则）+ **李沐《动手学深度学习》11 章**（优化器的从零实现）。

### 子模块 9.2：集成学习——三个臭皮匠顶个诸葛亮（西瓜书 §8.1—§8.5）

**学习目标**：理解两大范式——Boosting（串行，后一棵补偿前一棵的错误）vs Bagging（并行，多棵树投票）。西瓜书第 8 章是教材中最"工程化"的一章，直接对应 Kaggle 竞赛的核心武器 XGBoost/LightGBM。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **Boosting（AdaBoost）** | 按顺序训练——每次提高被上一次分错的样本的权重，强制新分类器关注"难样本" | §8.1 |
| **Bagging（Bootstrap Aggregating）** | 并行训练多棵树，每棵树用随机抽取（有放回）的子数据集——投票决定结果 | §8.2.1 |
| **随机森林** | Bagging + 每次分裂时随机选取一部分特征（不是全部特征），进一步增加树之间的多样性 | §8.2.2 |
| **结合策略** | 投票法（少数服从多数）、平均法（回归取均值）、学习法（Stacking——用另一个模型学怎么结合） | §8.3 |
| **多样性（Diversity）** | 集成有效的核心——个体学习器必须"好而不同"（准确率高于 50% 且彼此犯错不重合） | §8.4—§8.5 |

**实战任务**：
1. 同一份数据上对比：单棵决策树 vs Bagging（10棵） vs 随机森林（10棵） vs AdaBoost（10轮）——画出四个模型的决策边界在同一张 2×2 子图上
2. 用 sklearn 的 RandomForest 在手写数字数据集上训练，打印 feature importance，可视化哪些像素位置最重要
3. **加分项**：用 XGBoost 在加州房价数据集上做回归，调 `n_estimators`、`max_depth`、`learning_rate` 三个核心参数，理解它们分别控制什么

**🎯 首选资源**：

### 子模块 9.3：聚类与降维——没有标签怎么办（西瓜书 §9.1—§9.5 + §10.1—§10.6）

**学习目标**：无监督学习的两大任务——聚类（把数据自动分组）和降维（把高维数据压缩到低维同时保留最重要的信息）。

| 知识点 | 大白话解释 | 教材位置 |
|:---|:---|:---|
| **K-Means** | 随机选 K 个中心 → 分配每个点到最近的中心 → 重新计算中心 → 重复直到稳定 | §9.2 |
| **高斯混合模型（GMM）** | 比 K-Means 更灵活——每个簇是椭圆形的，每个点属于某个簇的概率（软分配） | §9.3 |
| **密度聚类（DBSCAN）** | 不需要事先指定 K！通过"密度相连"的概念自动发现簇——能发现任意形状的簇 | §9.5 |
| **PCA（主成分分析）** | 找数据方差最大的方向（第一主成分），然后找与之前方向正交的下一个最大方差方向…… | §10.3 |
| **k 近邻（kNN）** | 不需要训练！新样本的标签 = 离它最近的 k 个邻居的多数标签——懒惰学习（Lazy Learning） | §10.1 |
| **维数灾难** | 维度越高，数据变得越稀疏——高维空间中所有点都差不多远，传统距离度量失效 | §10.2 |
| **流形学习** | 数据虽然是高维的，但实际分布在一个低维的"流形"上（瑞士卷、S 形曲面） | §10.5 |

**实战任务**：
1. 手写 K-Means（纯 NumPy）：实现"随机初始化→分配→更新中心"循环，可视化每次迭代的簇中心移动轨迹（对应教材 §9.2）
2. **纯 NumPy 实现 PCA（用 SVD 分解）**：对 MNIST 784维 → 2维，画散点图，观察同一数字的样本聚在一起（对应教材 §10.3）
3. 用 PCA 做图像压缩：对一张彩图逐通道做 PCA，保留前 50 个主成分重建图像，计算压缩比和质量差异

**🎯 首选资源**：
- **西瓜书 §9.1—§9.5 + §10.1—§10.6 + 南瓜书第 9—10 章**。辅助动画：**3Blue1Brown《PCA 的本质》**。

### 🖥️ 硬件思维 Tip #8

> **思考题**：随机森林（Bagging）训练时，100 棵树之间没有任何依赖——你可以在 100 个 CPU 核心上同时训练（Embarrassingly Parallel）。但 Boosting 的训练是严格串行的——下一棵树必须等上一棵训练完。这两种计算模式在 GPU 上分别面临什么挑战？为什么深度学习（SGD 串行更新参数）反而在 GPU 上比随机森林快得多？（提示：深度学习每次迭代的计算量极大——矩阵乘法——GPU 用大规模并行掩盖了串行开销）

### 🖥️ 硬件思维 Tip #9

> **终极思考题**：Transfomer 的自注意力是 O(n²) 复杂度——n=128K 时注意力矩阵 = 32GB（远超 H100 的 80GB）。FlashAttention 不显式存储整个矩阵，分块计算让中间结果留在 GPU SRAM 中。查一下 GPU 的**显存层级**（HBM → L2 Cache → SRAM/Shared Memory → Register），理解为什么"数据局部性"是 AI 芯片设计的核心挑战。
